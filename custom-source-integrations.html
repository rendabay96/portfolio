Problem Statement

When an IT administrator of an organization integrates their Active Directory (AD) with Okta to enable employees with single sign-on for app access or to manage their user profile and lifecycle changes, they must first install one lightweight agent per AD instance which runs on the Windows host machine and keeps all changes between AD and Okta synchronized. Admins are instructed to keep these agents up-to-date as new versions containing bug fixes and new features are released.

Diagram of an Okta + AD integration. Many customers were operating on outdated AD agents.


Unfortunately, we discovered that the majority of our admins were not proactively updating their agents and would often wait until an Okta customer success manager prompts them to do so. We predicted it was due to the extremely manual process, but there were much more complicated reasons. My team and I set out to investigate how we might improve the Okta AD agent update workflow so that our users were not missing out on new functionalities.
